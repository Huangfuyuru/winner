### 1.使用变量  
sass使用$符号来表示变量
#### 1-1.变量声明  
`highlight-color:#F90`  
`$basic-border:1px solid black`  
当这个变量声明在{}内，我们不能在{}外使用这个变量，可以避免变量污染  
```
$nav-color:#F90;
nav{
    $width:100px;
    width:$width;
    color:$nav-color;
}
//编译后
nav{
    width:100px;
    color:#F90
}
```
#### 1-2.变量引用  
当改变变量的值时，所有引用此变量的地方生成的值都会随之改变  
```
$highlight-color:#F90;
$highlight-border:1px solid $highlight-color;
.selected{
    border:$highlight-border;
}
//编译后
.selected{
    border:1px solid #F90
}
```
### 2.嵌套CSS规则  
```
#content article h1{color:#333};
#content article p{margin-bottom:1.4em};
#content aside{background-color:#EEE};
//在scss中可以这样写
#content{
    article{
        h1{color:#333}
        p{margin-bottom:1.4em}
    }
    aside{
        background-color:#EEE
    }
}
记住：最外面的是父级，其他都是子级
```
#### 2-1.父选择器的标识符&  
他可以放在任何一个选择器可出现的地方,当包含父选择器标识符的嵌套规则被打开时，它不会像后代选择器那样拼接，而是&被父元素直接替换
```
article a {
    color:blue;
    &:hover{color:red}
}
//相当于
arcticle a {color:blue}
arcticle a:hover {color:blue}
```  
在为父级选择器添加:hover等伪类时，这种方式非常有用。  
#### 2-2.群组选择器的嵌套  
```
.container h1,.container h2,.container h3{
    margin-botton:.8em
}
//利用群组选择器的嵌套
.container{
    h1,h2,h3{
        margin-botton:.8em
    }
}

nav,aside{
    a{color:blue}
}
//内嵌套
nav a,aside a{
    color:blue
}
```
#### 2-3.子组合选择器和同层组合选择器:> + ~  
```
子组合选择器 >
article section{margin:5px}
article > section{margin:5px}
第一个会选择article下所有命中section选择器的子元素
第二个会选择article下子元素中最近命中section的元素  

同层相邻组合选择器 +  
header + p{font-size:1.1em}
header同层中紧跟的p元素

同层全体组合选择器 ~
article ~ article{border-top:1px}
article同层中所有article元素

//实例
article {
  ~ article { border-top: 1px dashed #ccc }
  > section { background: #eee }
  dl > {
    dt { color: #333 }
    dd { color: #555 }
  }
  nav + & { margin-top: 0 }
}
//解析后
article ~ article { border-top: 1px dashed #ccc }
article > footer { background: #eee }
article dl > dt { color: #333 }
article dl > dd { color: #555 }
nav + article { margin-top: 0 }
```
#### 2-4.嵌套属性  
在sass中，除了css选择器，属性也可以嵌套  
```
nav {
    border:{
        style:solid;
        width:1px;
        color:#CCC
    }
}
//解析
nav {
    border-style:solid;
    border-width:1px;
    border-color:#CCC
}
```
嵌套属性的规则:  
1.把属性名从中划线-的地方断开(border-style),在**根属性**后面添加一个冒号:  
2.然后紧跟一个{}块，把子属性部分写在{}中
```
nav {
    border:1px solid #ccc{
        left:0px;
        right:0px
    }
}
//解析
nav {
    border:1px solid #ccc;
    border-left:0px;
    border-right:0px;
}
```

@import "colors"  
导入colors.scss  
#### 3-1.使用SASS部分文件 
_开头 
#### 3-2.默认变量值  
!default  
含义是：如果这个变量被赋值了，那就用它声明的值，否则就用这个默认值  
```
$box-width:400px !default;
.fancybox{
    width:$box-width
}
```
如果用户在导入你的sass局部文件之前声明了一个$box-width变量，那么你的局部文件中对$box-width赋值400px的操作就无效，如果用户没有做这样的声明，则$box-width将默认为400px
#### 3-3.嵌套导入  
sass允许@import命令写在css规则内，这种导入方式，生成对应的css文件时，局部文件会被直接插入到css规则内导入他的地方   
一个名_blue-theme.css局部文件
```
aside{
    background:blue;
    color:white
}
``` 
然后把他导入到一个css规则内  
```
.blue-them{@import "blue-theme"}
//解析
.blue-them{
    aside{
        background:blue;
        color:white
    }
}
```
#### 3-4.原生CSS导入  
以下三种情况会生成原生的CSS @import  
* 被导入文件的名字以.css结尾  
* 被导入文件的名字是一个url地址（比如http://www.sass.hk/css/css.css）  
* 被导入文件的名字是css的url值  
这就是说，你不能用sass的@import直接导入一个原始的css文件，因为sass会认为你想用css原生的@import。但是，因为sass的语法完全兼容css，所以你可以把原始的css文件改名为.scss后缀，即可直接导入了。

### 5.混合器  
使用场景:样式变的越来越复杂，需要大段的重用样式的代码，可以通过混合器实现大段样式的重用  
混合器使用@mixin标识符定义，@include调用   
* 定义一个混合器 
```
@mixin rounded-corners{
    -moz-border-radius: 5px;
    -webkit-border-radius: 5px;
    border-radius: 5px;
}
```  
* 调用这个混合器  
```
notice {
  background-color: green;
  border: 2px solid #00aa00;
  @include rounded-corners;
}
//解析
.notice {
  background-color: green;
  border: 2px solid #00aa00;
  -moz-border-radius: 5px;
  -webkit-border-radius: 5px;
  border-radius: 5px;
}
```
#### 5-1.何时使用混合器  
* 能否为这个混合器想出一个好名字 rounded-corners
#### 5-2.混合器中的CSS规则  
混合器中不仅可以包含属性，也可以包含css规则，包含选择器和选择器中的属性  
```
@mixin no-bullets {
    list-style:none;
    li{
        list-style-image:none;
        list-style-type:none;
        margin-left:0px
    }
}
```  

```  
ul.plain{
    color:#444;
    @include no-bullets;
}
```  
```
ul.plain{
    color:#444;
    list-style:none;
}
ul.plain li{
    list-style-image:none;
    list-style-type:none;
    margin-left:0px
}
```
#### 5-3.给混合器传参  
可以通过在@include混合器时给混合器传参，来定制混合器生成的精确样式。  
```
@mixin link-colors($normal,$hover,$visited){
    color:$normal;
    &:hover{color:$hover}
    &:visited{color:$visited}
}
```
```
a{
    @include link-colors(blue,red,green)
}
//也可以用
a{
    @include link-colors(
        $normal:blue,
        $visited:green,
        $hover:red
    )
}
这样就不用指定传参顺序了
```
生成  
```
a{color:blue}
a:hover{color:red}
a:visited{color:green}
```
#### 5-4.默认参数值  
```
@mixin link-colors(
    $normal,
    $hover:$normal,
    $visited:$normal
){
    color:$normal;
    &:hover{color:$hover}
    &:visited{color:$visited}
}
```
### 6.使用选择器继承来精简CSS  
选择器继承：一个选择器可以继承另一个选择器定义的所有样式 **@extend**  
```
.error{
    border:1px solid red;
    background-color:pink
}
.seriousError{
    @extend .error;
    border-width:3px
}
```
.seriousError会继承.error定义的所有样式  
.seriousError不仅会继承.error自身的所有样式，任何跟.error有关的组合选择器样式也会被.seriousError以组合选择器的形式继承  
```
.error a{  //应用到.seriousError a
    color:red;
    font-weight:100
}
h1.error{ //应用到h1.seriousError
    font-size:1.2rem
}
```
